1. GRE (Guest Relations Executive)

First Point of Contact
📞 Guest Enquiry Arrives (Walk-in / Call / Referral)

1 Open Add Party Page

  • Route: /add-party
  • GRE sees simplified form
  • Auto-loads draft from localStorage
→

2 Fill Required Fields

  • Date (or TBC: Month Year) *
  • Host Name *
  • Phone Number (Indian: 10 digits) *
  • Company Name *
  • Expected Pax (e.g. 50 or 40-60) * GRE only
→

3 Fill Optional Fields

  • Occasion Type (Corporate/Family/Others)
  • Guest Visited (Yes/No)
  • Place / Venue
  • Meal Type (e.g. Lunch 12:30 PM)
  • Alt Contact
  • Special Requirements
  • Remarks
  • Handled By (dropdown: Sales/Manager users)
↓

4 Validations Run

  • Phone format validated (Indian)
  • Duplicate check: Same phone + date
  • If duplicate found: warning shown (not blocked)
  • Past date prevention: min={today}
  • Expected Pax required for GRE
→

5 Submit (POST /api/parties)

  • Auto-generates Unique ID (AKN-XXXXXX)
  • Status auto-set to ENQUIRY
  • Sets Enquired At timestamp
  • Auto-calculates Day from Date
  • Writes to Google Sheets
  • Draft cleared from localStorage
→

6 Auto Notifications

  • Email: New Party Notification sent to SALES + MANAGER
  • In-App: Bell notification created (type: new_enquiry)
  • WhatsApp: Copy message button available
👁 GRE Can Also Do

View Dashboard

  • See all parties (all statuses)
  • NO financial stats visible
  • NO revenue data
  • Auto-refresh every 30 seconds

View Calendar

  • Color-coded party view by month
  • View-only (cannot click to edit)
  • Status counts in legend

View Party Detail

  • Can view all party info
  • Cannot Edit (no edit button)
  • Can copy WhatsApp message

3. SALES - Full Party Management

Core Workflow
🖂 Step A: Contact the Guest

A1 View Enquiry on Dashboard

  • See new enquiry in party list
  • Click to open Party Detail
  • Check stale enquiry popup alerts
  • Review: Host, Phone, Company, Pax, Requirements
→

A2 Contact Guest

  • Call / WhatsApp / Email
  • Discuss: Packages, Menu, Pricing, Dates
  • Copy WhatsApp message (auto-generated)
→

A3 Update Status

  • Change: ENQUIRY → CONTACTED
  • API: PUT /api/parties/:id/status
  • Auto-adds follow-up note with timestamp
  • Auto-updates Last Follow Up Date
  • Email: Status Change Notification sent
↓
💬 Step B: Negotiate & Follow Up

B1 Add Follow-Up Notes

  • API: PUT /api/parties/:id/followup
  • Notes prepended with: [date - username]
  • Updates Last Follow Up Date
  • Multiple follow-ups tracked chronologically
→

Guest Interested?

  • YES → Continue to Step C
  • NO → Cancel Party
→

If Cancelled

  • Status: CANCELLED
  • Lost Reason required (mandatory)
  • Cancellation Notification email sent to MANAGER + ADMIN
  • Cancelled Date auto-set
  • Flow ends here
↓
📋 Step C: Mark Tentative & Prepare F&P

C1 Status → Tentative

  • Change: CONTACTED → TENTATIVE
  • Guest is interested but not confirmed yet
  • Status change notification email sent
  • Follow-up note auto-added
→

C2 Update Party Details

  • Confirm: Date, Pax, Meal Type
  • Set: Package Selected
  • Set: Approx Bill Amount
  • Set: Guest Email
  • Update: Special Requirements
  • Handled By auto-appended
→

C3 Create F&P Document

  • Navigate to F&P → Create New
  • Select package, menu items, addons
  • Set rate/head, programme schedule
  • Generate PDF & email to guest
  • Sets FP Issued = "Yes" on party
  • (See Section 4 for full F&P flow)
↓
✅ Step D: Confirm Booking

D1 Status → Confirmed

  • Change: TENTATIVE → CONFIRMED
  • OR Auto-confirmed when advance payment received
  • Status change notification sent
  • Now visible to CASHIER & ACCOUNTS
→

D2 Set Confirmed Details

  • Confirmed Pax
  • Final Rate / Head
  • Final Total Amount (auto: Pax x Rate)
  • Guest Email
  • Balance Payment Date
→

D3 Collect Advance Payment

  • API: PUT /api/parties/:id/payment
  • Type: "advance"
  • Records: Amount, Method, Reference, Note
  • Auto-calculates: Total Advance, Due Amount
  • Payment Status: Unpaid → Partial/Paid
  • Billing Update email sent
🛠 Additional Sales Actions

Send Payment Reminder

  • API: POST /api/parties/:id/send-payment-reminder
  • Requires: Guest Email + Due > 0
  • Sends reminder email to guest
  • Logged in Reminder Log

Edit Party Fields

  • Date, Host, Phone, Company, Status
  • Place, Meal, Pax, Remarks, Occasion
  • Package, FP Issued, Lost Reason
  • All billing fields
  • Follow-up Notes, Guest Email

View Reports

  • Daily summary reports
  • Date range reports
  • Financial overview
  • Status analysis charts

6. CASHIER - Billing & Payments

Confirmed Parties Only
💳 Billing Workflow

1 Lookup Party

  • Route: /cashier-billing
  • Search by Unique ID (AKN-XXXXXX)
  • API: GET /api/parties/lookup/:uniqueId
  • Duplicate warning if same phone+date exists
  • Only CONFIRMED parties shown
→

2 Review & Set Billing

  • Confirmed Pax
  • Final Rate / Head
  • Final Total Amount (auto: Pax x Rate)
  • Bill Order ID
  • Guest Email
  • Balance Payment Date
  • Validation: Rate required for confirmed parties
  • Validation: If due > 0, Balance Date + Email required
→

3 Record Payments

  • API: PUT /api/parties/:id/payment
  • Type: "advance" or "payment"
  • Each entry records: Amount, Method, Reference, Note, Timestamp, User
  • Appended to Payment Log (JSON array)
  • Multiple entries supported
↓

4 Auto-Calculations

  • Total Advance Paid = Sum of advance entries
  • Total Paid = Sum of payment entries
  • Total Amount Paid = Advance + Paid
  • Due Amount = Final Total - Total Amount Paid
→

5 Payment Status (Auto-Set)

  • UNPAID = Total Paid is 0
  • PARTIAL = 0 < Paid < Total
  • PAID = Paid ≥ Total
→

6 Billing Notification

  • Billing Update email sent to MANAGER + ADMIN + ACCOUNTS
  • Contains: Payment details, total bill, due amount
  • If advance on non-confirmed: auto-confirms party

9. Auto-Calculations

Real-time

Final Total Amount

Confirmed Pax x Final Rate/Head
Falls back to Expected Pax if Confirmed not set.
Range "40-60" uses minimum (40).

Payment Totals

Total Advance = sum of advance entries
Total Paid = sum of payment entries
Total Amount Paid = Advance + Paid

Due Amount

max(0, Final Total - Total Amount Paid)
Never goes negative.

Payment Status

Unpaid: Total Paid = 0
Partial: 0 < Paid < Total
Paid: Paid ≥ Total

Day from Date

YYYY-MM-DD → "Monday", "Tuesday" etc.
TBC dates → "TBC"
Timezone-safe manual parsing.

Approx Balance

max(0, Approx Bill - Total Amount Paid)
For estimation before final billing.

11. All Email Notifications

12 Types

1. New Party Notification

Trigger: Party created. To: SALES, MANAGER. Content: Date, host, phone, company, pax, package, status.

2. Status Change

Trigger: Status updated. To: MANAGER, SALES. Content: Old → New status, party details.

3. Cancellation

Trigger: Party cancelled. To: MANAGER, ADMIN. Content: Lost reason, cancelled date.

4. Stale Enquiry Alert

Trigger: Enquiry >1hr old (every 15min). To: SALES, MANAGER. Content: Table of stale parties.

5. Critical Alert

Trigger: Same as stale (escalation). To: ADMIN only. Content: Handled By, hours pending.

6. Daily Follow-up

Trigger: 9 AM daily. To: SALES, MANAGER, ADMIN. Content: Follow-ups + pending payments + upcoming parties.

7. Daily Night Report

Trigger: 10 PM daily. To: MANAGER, ADMIN. Content: Day's statistics, revenue, status counts.

8. Billing Update

Trigger: Payment added. To: MANAGER, ADMIN, ACCOUNTS. Content: Payment details, total, due.

9. Pending Payments (ACCOUNTS)

Trigger: 9:30 AM daily. To: ACCOUNTS, MANAGER. Content: Table of all parties with outstanding balance.

10. Payment Reminder (Guest)

Trigger: 8 AM on due date / manual. To: Guest email. Content: Bill, paid, balance, urgency banner.

11. F&P Email

Trigger: Manual send. To: Custom email. Content: Event details, formatted menu selections table.

12. Test Email

Trigger: Manual (Manager). To: Configured email. Content: SMTP connectivity verification.

14. All Application Pages

Frontend Routes

/login

Username/password auth. JWT token stored. Redirect to dashboard. Auto-deactivates inactive users (2+ months).

/ (Dashboard)

Party list with filters (status, date, search, type). Stats cards. Quick actions (Confirm, Cancel, Payment). Stale enquiry popup. Follow-ups section. Auto-refresh 30s.

New enquiry form. Auto-draft save. Duplicate detection. Phone validation. Past date prevention. GRE simplified form. WhatsApp copy.
